Some of these situations were touched on in the switch statement discussion. 1.2 Response

There are numerous selection structures available for use in modern C++, and for the most part they are interchangable. There are certain situations in which certain structures will perform better than others.

1.2.1 if Statements

If one wishes to select which statement to execute based on one simple condition, this is called a one-way selection (Malik, 2015,  §4-1d) . When attempting to select a statement based on a single condition (or a compound expression) while assigning a statement to execute if the conditions are not met, we use a two-way selection structure, “if...else...” (Malik, 2015,  §4-1e) . When we are required to incorporate multiple selections, we use a nested “if...else...” structure (Malik, 2015,  §4-1l) .

Example (One-Way Selection). Suppose a piece of software wanted to ensure users were authenticated in order to visit a certain route on the server. This could be checked by writing an “if” statement as provided in Listing 1.1 .

Example (Multi-Way Selection). Suppose a game wanted to dictate what strength level to give its users based on his or her level in the game. If their level is 100, the user gets ∞ assigned to their strength. If the user is between levels 50 and 99, their strength is set to 1000. If the user is between level 25 and 49, their strength is set to 500. For levels < 25, we will assign 250 as their strength value. To assign the correct strength value to the user, we can use the nested if...else... statement as described in Listing 1.3 .

// Note: I am aware that $\infty$ is not a valid value.

if (user.level >= 100) {
  user.strength = $\infty$;
  else {

    } else {

          user.strength = 250;
        }
      }
    }
  }
}

The deciding factor between using a nested if...else... statement versus a series of if statements lies in that when using the series of statements, more than one statement could execute if multiple conditions are true. Using the nested if...else... statement allows us to prevent that from happening by only executing the ﬁrst statement which matches the given constraints.

1.2.2 switch Structures

Suppose we have a lot of alternatives for selection in our program. Nested if...else... structures tend to get a bit heavy when using a lot of selection options, which is where C++‘s switch statement comes in. First the expression is evaluated and that is used to select amongst the alternatives given in the statements following the reserved word, “case”. See the below example and Listing 1.4 for basic syntax.

Example (Switch Syntax). Suppose you wanted to alert users as to which group they belong to, admin, user, or guest. To do this in C++, we would use the basic snippet of a switch structure as seen in Listing 1.4 .

    cout << "You are in the guest group." << endl;
    break;
}
